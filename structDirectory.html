Detailed Description

Directory monitoring for hot-reload functionality.

Monitors a directory for file changes using Windows ReadDirectoryChangesW API. Triggers reload events when Lua scripts are modified.

Static class for directory change monitoring

Runs in a dedicated thread to monitor file system changes without blocking the main application thread.

Parameters
directoryPathDirectory to monitor (wide string)
eventDispatcherEvent dispatcher for change notifications

Continuously monitors the specified directory for file modifications and creation/deletion events. When changes are detected, dispatches "OnDirectoryChange" event.

Note
Runs in an infinite loop - should be executed in a separate thread
Only monitors for FILE_NOTIFY_CHANGE_LAST_WRITE and FILE_NOTIFY_CHANGE_FILE_NAME events
Warning
Uses blocking I/O - ensure directory path is valid
High-frequency file changes may cause event storms
