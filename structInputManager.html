Detailed Description

Input simulation for keyboard and mouse.

Provides cross-platform input simulation using Windows SendInput API. Supports keyboard events, text typing, and mouse operations.

Static methods for simulating user input

All methods are thread-safe for use from Lua callbacks. Input is simulated at system level, not application level.

Move mouse cursor to absolute screen coordinates.

Parameters
xX coordinate (0 = left edge, screen width = right edge)
yY coordinate (0 = top edge, screen height = bottom edge)

Moves cursor instantly to specified position using absolute coordinates. Coordinates are transformed to Windows' virtual screen space (0-65535).

Note
Uses MOUSEEVENTF_MOVE | MOUSEEVENTF_ABSOLUTE flags
Coordinates outside screen bounds are clamped by Windows
mouse_move(1920, 1080) // Move to bottom-right corner of 1080p screen
mouse_move(0, 0) // Move to top-left corner

Parameters
vkVirtual key code (use KEY constants)

Simulates pressing and immediately releasing a key. Equivalent to a quick tap on the keyboard.

Note
Uses Windows SendInput() with KEYDOWN and KEYUP events
For modifier keys, consider holding them with multiple calls
send(KEY.ENTER) // Press and release Enter
send(KEY.ESCAPE) // Press and release Escape

Type text as if from keyboard.

Parameters
textString to type

Simulates typing complete strings with proper shift key handling for uppercase and special characters. Includes small delays between keystrokes for reliability with some applications.

Note
Handles shift states for characters like @, !, %, etc.
Includes 15ms delay between keystrokes
Not suitable for extremely fast typing simulation
write("Hello, World!") // Types with proper shift for '!' and 'H'
write("test@example.com") // Handles @ symbol with shift
